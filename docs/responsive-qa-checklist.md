# PR-41 반응형 QA 체크리스트

PR-41은 기존 QuoteOps AI 프론트엔드 화면을 다시 설계하지 않고, 모바일과 태블릿에서 읽기 쉽고 조작 가능한 상태로 다듬는 단계다. 백엔드 로직, 가격 계산식, 인증, 데이터베이스, API 계약은 변경하지 않는다.

## 확인 범위

- desktop 1200px 이상: 넓은 화면에서 내비게이션, 카드, 표, 결과 패널이 과하게 늘어나지 않는지 확인한다.
- laptop 1024px-1199px: 내비게이션과 카드 그리드가 2-3열로 안전하게 줄어드는지 확인한다.
- tablet 768px-1023px: 업무 화면, 폼, 결과 패널이 1-2열로 전환되고 터치 대상이 충분한지 확인한다.
- mobile 768px 미만: 한 줄 단일 컬럼 중심으로 표시되고 페이지 전체에 가로 넘침이 없는지 확인한다.

## 화면별 확인 대상

- 홈
- 견적
- 고객 요청
- 가격
- 승인
- 시뮬레이션
- 분석
- 리포트
- 운영
- 데모

## 수동 브라우저 QA

- 페이지 전체에 가로 넘침이 없다.
- navigation 항목은 줄바꿈되어도 읽을 수 있고 활성 상태가 보인다.
- cards와 card grids는 태블릿/모바일에서 좁게 찌그러지지 않고 아래로 쌓인다.
- forms는 작은 화면에서 단일 컬럼으로 쌓이고 input, select, textarea가 컨테이너를 넘지 않는다.
- table-wrap 안의 표만 가로 스크롤되고 전체 페이지 폭은 유지된다.
- buttons와 action groups는 줄바꿈되거나 모바일에서 full-width로 쌓인다.
- status badges와 chips는 카드 폭을 밀어내지 않고 여러 줄로 감긴다.
- login/demo panels는 모바일에서 화면을 깨지 않고 주요 액션을 누를 수 있다.
- 비어 있음, 오류, disabled 상태는 12px 이상 크기로 읽을 수 있다.

## Render 재배포 확인

PR-42에서 Render 배포 URL 기준으로 홈, 견적, 가격, 승인, 분석, 리포트, 운영, 데모 화면을 다시 열어 blank page와 가로 스크롤 발생 여부를 최종 확인한다.
